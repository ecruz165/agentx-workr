ACCOUNTABILITY

Productaccountable for feature intent
Tech Leadaccountable for technical approval
Human Approveraccountable for gate decision
Agentproduces execution status + evidence
GATES:
● Continue — output sufficient
● Auto-refine — agent self-corrects
● Escalate — human re-enters — scope ends at develop
⚠️
Scope ends at merge to develop. Release, production rollout, rollback, and compliance approvals are downstream and out of scope.
KEY PRINCIPLE
Humans govern meaning. Tech design produces execution-safe artifacts. Maintenance agent identifies code health issues. Agents execute within approved scope. · Product accountable for feature intent · Tech Design accountable for technical approval · Human approver accountable for gate decision · Agent produces execution status + evidence. Phase 3 runs as one Team Agent workspace — sub-agents work across all repos simultaneously. Both feature work (Phase 1→2) and maintenance work (Phase M) converge on Phase 3 after HITL approval. Escalations route back to Phase 2 / Phase M or are resolved inline via HITL. 3b assembles the combined preview and merges all PRs to develop.
